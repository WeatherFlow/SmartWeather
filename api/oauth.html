Overview

The Smart Weather API supports the Authorization Code grant type. For clients that are unable to maintain the confidentiality of the client secret, the API also supports the Authorization Code with PKCE grant type.

Creating a Client Application

Before you can begin using OAuth with the Smart Weather API, you must register your application with WeatherFlow. To do so, please contact us.

Authorization Code Grant

Step 1:

Request authorization from the user using the authorization endpoint below.

https://smartweather.weatherflow.com/authorize.html

With the request include the following query string parameters:

Query String Parameter Description
client_id Provided to you when registering your application
response_type Set this value to code to indicate that you would like an authorization code returned.
redirect_uri The url you want the user to be redirected to after the authorization is completed. This url must be registered with WeatherFlow.

Step 2:

Once redirected to the authorization page, the user will approve or deny the authorization request. If they approve the request they will be redirected back to the redirect URL you provided along with an authorization code.

Exchange the authorization code for an access token by making a POST request to the API endpoint below.

https://swd.weatherflow.com/id/oauth2/token

In the URL Encoded Form body of the POST request include:

Form Field Description
grant_type Set this value to authorization_code
code Send the authorization code received from the authorization server. This is the code
client_secret Send the client secret provided when your application was created.

If you are not using a server application and cannot properly secure the client secret use the Authorization Code Grant with PKCE method.
